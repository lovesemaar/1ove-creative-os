# License Options

This package intentionally ships with `LICENSE_PENDING.md` instead of a final license because the creator must choose the release strategy.

## Option 1: MIT

Best for maximum adoption and simple reuse. Others can use, modify, and commercialize with attribution and license notice.

## Option 2: Apache 2.0

Best for permissive release with more detailed patent language. Good for serious open-source software projects.

## Option 3: AGPLv3

Best if you want modifications and hosted/network versions to remain open under similar terms.

## Option 4: Dual License

You may release code under one license and creative knowledge assets under another.

## Recommendation Before Public Release

Use `LICENSE_PENDING.md` until you decide:

- how open you want the software to be,
- whether brand/identity documents should be reusable,
- whether commercial reuse should be allowed,
- whether public modifications should remain open.

# Privacy Policy for Local Prototype

This project is designed as a local-first creative OS.

## Default Principle

Private knowledge should guide creative direction only when explicitly allowed. Public outputs should use symbolic, cinematic language instead of exposing private trauma, diagnosis details, real names, or sensitive personal memories.

## Memory Tiers

- `public` — safe brand rules, public themes, reusable creative facts.
- `project` — project decisions, workflow notes, build history.
- `private` — personal notes that should not be used in public prompts by default.
- `sensitive` — highly private material that must stay blocked unless manually and intentionally enabled by the developer.

## Public-Safe Language

Use terms such as:

- duality
- inner conflict
- survival identity
- emotional split
- guilt
- darkness
- humanity
- healing
- integration
- reconstruction

## Never Auto-Expose

Do not automatically expose:

- diagnosis details
- trauma specifics
- real names
- addresses
- phone numbers
- private relationship details
- sensitive memory entries
- medical/legal/financial private data
